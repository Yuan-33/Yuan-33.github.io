Eagle-v2

Speculative Decoding — Dynamic Draft Tree
Eagle-v2 在 Eagle-v1 的基础上引入了动态 Draft Tree 结构，根据每个 Draft token 的置信度分数自适应调整树的宽度和深度，将计算资源集中在高概率路径上。这一改进使 Eagle-v2 的接受率和端到端加速比进一步提升，成为 2024 年投机解码的 SOTA 方案。

核心改进

Eagle-v1 使用固定的树结构进行 Draft，所有分支的宽度和深度相同。Eagle-v2 发现不同位置的 Draft token 置信度差异很大——高置信度 token 更可能被接受，应该扩展更深；低置信度 token 应该被剪枝。Eagle-v2 根据 Draft Head 输出的概率动态构建树，实现了更高效的计算分配。

关键特性

  • 动态树构建：根据 Draft token 概率动态调整树的形状，高概率路径更深更宽，低概率路径提前终止，最大化有效 Draft 数量
  • Tree Attention 验证：将整个 Draft Tree 组织为一个批次，通过一次前向传播验证所有分支，充分利用 GPU 并行能力
  • 接受率提升 20-40%：相比 Eagle-v1 的固定树结构，动态树在各类任务上平均接受率提升 20-40%
  • 可配置树大小：支持设置最大树节点数，在延迟与吞吐之间灵活权衡，适应不同部署场景
  • 端到端加速：在 Vicuna-7B/13B/70B 上实现 3.0-4.0x 加速，在长文本生成场景下加速比更高
