Eagle-v3

Speculative Decoding — Context-Aware Optimization
Eagle-v3 彻底重构 Draft Model 训练与推理范式，通过移除特征约束、引入训练时模拟测试和多层特征融合，彻底消除训练推理分布差异，大幅释放小模型预测能力，成为当前最接近理论最优的投机解码方案。

设计哲学与核心改进

Eagle-v3 没有在树结构或推理策略上小修小补，而是从**训练目标、特征输入、分布对齐**三个底层维度重构了 Draft Model。相比 v1/v2，它做了两个颠覆性改动，彻底解决了传统特征模仿带来的性能瓶颈。

1. 移除特征约束与 Training-Time Test

Eagle-v1/v2 都使用特征回归损失 lfea，强制 Draft Model 模仿主模型（Target Model）的顶层特征。这种方式虽然稳定，但严重限制了 Draft Model 的表达空间。

Eagle-v3 直接移除特征回归损失，仅保留 Token 分类损失。Draft Model 不再需要对齐主模型特征，只需要对最终生成的 Token 负责，在高维特征空间中可以自由优化。

但移除 lfea 会引发致命问题：训练与测试分布不一致。训练时使用 Teacher Forcing（喂真实特征），推理时只能使用自身生成的特征，输入分布差异会导致模型从第二步开始性能崩溃。

解决方案：Training-Time Test（训练时模拟测试）
在训练过程中直接模拟推理行为，实现分布对齐：
Step 1：输入主模型特征 ft，Draft Model 生成 ât+1
Step 2：下一步输入不再使用真实特征 ft+1，而是直接使用上一步生成的 ât+1
让 Draft Model 在训练时就适应自身生成的带误差输入，完全消除训练/推理 Gap。

2. 多层特征融合

移除特征约束后，Draft Model 不再局限于使用主模型顶层特征。Eagle-v3 创新性地从主模型抽取低、中、高三层语义特征进行融合：

  • Low-level：底层语义（语法、结构信息）
  • Middle-level：中层语义（短语、局部依赖）
  • High-level：高层语义（全局逻辑、意图）

融合公式：
g = FC( Concat( l, m, h ) )

通过拼接 + 全连接层降维，Draft Model 获得远超单一顶层特征的语义信息，预测准确率大幅提升。

3. 动态 Draft Tree 继承与优化

Eagle-v3 沿用 v2 的动态 Draft Tree结构，根据置信度动态生成树形结构，而非固定链式生成。结合更强的 Draft Model，树的有效分支比例、接受率都达到新高度。

关键技术实现

  • 移除特征回归损失：放弃模仿主模型特征，Draft Model 只对 Token 预测负责，表达能力完全释放
  • Training-Time Test：训练时直接使用自身生成特征作为输入，彻底对齐训练/推理分布
  • 多层特征融合：融合主模型低/中/高三层特征，语义信息更丰富，预测更精准
  • 轻量 Draft 结构：Draft Model 为单层 Transformer Decoder，计算开销极低
  • 稀疏对角 Attention Mask：保证因果性，高效模拟 Draft Tree 生成流程
  • 动态 Draft Tree：置信度驱动树形结构，无冗余计算，接受率逼近理论上限

面试常问 Q&A

Q1：Eagle-v3 为什么要移除特征回归损失 lfea？

A：v1/v2 强制 Draft 模型对齐主模型特征，虽然稳定但限制了表达能力。Eagle-v3 只保留 Token 损失，让 Draft 模型在高维空间自由优化，专注于最终预测效果，释放全部潜力。

Q2：什么是 Training-Time Test？解决了什么问题？

A：训练时模拟推理行为，下一步输入直接使用 Draft 模型上一步生成的特征，而不是主模型真实特征。它彻底解决了“训练用真实特征、推理用生成特征”的分布不一致问题，避免模型在推理时崩盘。

Q3：多层特征融合相比只用顶层特征好在哪里？

A：顶层特征只包含抽象语义，浅层包含语法结构，中层包含局部依赖。融合后 Draft Model 能同时感知全局逻辑和局部细节，预测更准确，接受率显著提升。

Q4：Eagle-v3 的 Draft Model 结构是什么？

A：标准单层 Transformer Decoder Layer。结构轻量、推理极快，配合特殊的稀疏对角 Attention Mask 实现训练时的序列模拟。

Q5：Eagle 系列 v1→v2→v3 的核心演进是什么？

A：v1 提出特征空间 Draft；v2 提出动态 Draft Tree 减少计算浪费；v3 从底层训练机制突破，移除特征约束+训练测试对齐+多层特征融合，达到理论最优性能。
