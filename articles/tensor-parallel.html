    Tensor Parallelism

    Parallel Strategy — Intra-Layer Sharding
    Tensor Parallelism（TP）是将单个 Transformer 层的权重矩阵按列或行切分到多个 GPU 上并行计算的策略。每个 GPU 只持有权重的一部分，通过 AllReduce 通信聚合结果。TP 适用于节点内多 GPU 场景（高带宽 NVLink/NVSwitch），是 Megatron-LM 提出的核心并行策略，也是大模型推理中最常用的并行方式。

    核心原理

    大模型的参数量通常远超单个 GPU 的显存容量。例如 LLaMA-2-70B 有 70B 参数，FP16 下需要 140GB 显存，而单个 A100 只有 80GB。Tensor Parallelism 通过将每个层的权重矩阵切分到多个 GPU 上，使得每个 GPU 只需存储和计算权重的一部分，从而支持在多 GPU 上运行超大模型。

    列并行（Column Parallel）与行并行（Row Parallel）

    Transformer 层中的核心计算是矩阵乘法 Y = XW，其中 X 是输入，W 是权重矩阵。TP 将 W 按列或按行切分：

    列并行（Column Parallel）：将权重 W 按列切分为 [W1, W2, ..., Wn]，每个 GPU i 持有 Wi，独立计算 Yi = XWi。优点是各 GPU 可以完全独立计算，无需通信。但输出 Yi 只是最终结果的一部分，需要后续处理。

    行并行（Row Parallel）：将权重 W 按行切分为 W = [W1; W2; ...; Wn]，同时将输入 X 也按相应维度切分。每个 GPU i 计算 Yi = XiWi，最终通过 AllReduce 求和得到完整结果 Y = ΣYi。行并行需要在计算前对输入进行切分（Scatter），计算后对输出进行聚合（AllReduce）。

    MLP 层的并行方案

    Transformer 的 MLP 层包含两个线性变换：Y = GeLU(XW1)W2。Megatron-LM 的方案是：W1 使用列并行，W2 使用行并行。具体地：（1）W1 按列切分到 2 个 GPU，每个 GPU 独立计算 GeLU(XW1_i)，GeLU 是逐元素操作，可以独立执行；（2）W2 使用行并行，每个 GPU 计算部分结果，然后 AllReduce 聚合。这种「列-行」交替的方案使得 MLP 层只需要一次 AllReduce 通信。

    Attention 层的并行方案

    Multi-Head Attention 天然适合 TP——将不同的 Attention Head 分配到不同的 GPU 上。具体地：（1）QKV 投影矩阵按列切分，每个 GPU 持有一部分 Head 的 Q/K/V；（2）各 GPU 独立计算其 Head 的 Attention；（3）输出投影矩阵按行切分，各 GPU 计算部分结果后 AllReduce 聚合。对于 GQA/MQA 模型，KV Head 数量少于 GPU 数量时，需要多个 GPU 共享同一个 KV Head，通信模式略有不同。

    通信开销分析

    TP 的主要通信开销是 AllReduce——每层需要 2 次 AllReduce（MLP 一次 + Attention 一次）。AllReduce 的通信量为 2 × (TP-1)/TP × hidden_dim × batch_size × seq_len × dtype_bytes。在 NVLink（600GB/s on A100）下，这个通信开销可以被计算掩盖（通信与计算重叠）。但在 PCIe（64GB/s）或跨节点（25-100Gb/s）下，通信开销会成为瓶颈。因此 TP 通常限制在单个节点内（TP=8 对应 8 GPU 节点），跨节点使用 Pipeline Parallelism。

    关键特性

    • 层内切分：将单个层的权重矩阵按列/行切分到多个 GPU，每个 GPU 只计算权重的一部分
    • 低延迟：TP 的通信是 AllReduce，在 NVLink 下延迟极低（微秒级），适合推理的低延迟需求
    • 适用于节点内：TP 依赖高带宽互联（NVLink/NVSwitch），通常限制在单节点 8 GPU 内
    • Head 级并行：Attention 层天然支持 TP——将不同 Head 分配到不同 GPU，无需额外切分
    • 近线性加速：在 NVLink 下，TP=8 的加速比约为 7-7.5x（通信开销约 5-10%）
    • 行业标准：Megatron-LM、vLLM、TensorRT-LLM、DeepSpeed 均原生支持 TP

    面试常问 Q&A

    Q1：Tensor Parallelism 和 Data Parallelism 的区别是什么？

    A：Data Parallelism（DP）是每个 GPU 持有完整的模型副本，处理不同的数据批次，通过 AllReduce 同步梯度（训练）或直接独立推理。TP 是每个 GPU 只持有模型的一部分（层内切分），所有 GPU 协同处理同一批数据。DP 适合模型能放入单 GPU 的场景，TP 适合模型太大需要多 GPU 的场景。实践中两者经常组合使用——例如 TP=8（节点内）× DP=4（跨 4 个节点）。

    Q2：为什么 TP 通常限制在单节点内？

    A：TP 每层需要 2 次 AllReduce 通信，通信频率非常高。NVLink 的带宽为 600GB/s（A100）或 900GB/s（H100），可以将通信延迟控制在微秒级，与计算重叠。但跨节点的网络带宽通常只有 25-100Gb/s（InfiniBand），比 NVLink 低 50-100 倍，通信延迟会成为严重瓶颈。因此 TP 通常限制在单节点 8 GPU 内，跨节点使用 Pipeline Parallelism（通信频率低得多）。

    Q3：Megatron-LM 的列并行+行并行方案是如何工作的？

    A：对于 MLP 层 Y = GeLU(XW1)W2：（1）W1 按列切分，每个 GPU 独立计算 GeLU(XW1_i)，无需通信；（2）W2 按行切分，每个 GPU 计算部分结果后 AllReduce 求和。这种「列-行」交替使得 MLP 层只需一次 AllReduce。对于 Attention 层：QKV 按列切分（不同 Head 分配到不同 GPU），输出投影按行切分，同样只需一次 AllReduce。每个 Transformer 层总共 2 次 AllReduce。

    Q4：GQA/MQA 模型的 TP 如何处理？

    A：GQA/MQA 的 KV Head 数量少于 Query Head 数量。在 TP 中，Query Head 可以按 GPU 数量切分（例如 32 个 Head 分给 8 个 GPU，每个 GPU 4 个 Head）。但 KV Head 只有 8 个（GQA），如果 TP=8 则每个 GPU 恰好 1 个 KV Head；如果 TP=4 则每个 GPU 2 个 KV Head。当 KV Head 数量少于 TP 度数时（如 MQA 只有 1 个 KV Head），需要通过 AllGather 通信将 KV Head 广播到所有 GPU，这会增加通信开销。

    Q5：TP 对推理的 batch size 有什么影响？

    A：TP 将权重切分到多个 GPU，每个 GPU 的显存占用减少，可以容纳更大的 batch size。例如 LLaMA-2-70B 在 TP=1 时需要 140GB 显存（无法放入单 GPU），TP=8 时每个 GPU 只需约 17.5GB 权重 + KV Cache 显存，可以服务大量并发请求。但 TP 也意味着每个 GPU 的 KV Cache 容量按比例减少（TP=8 时每个 GPU 只有 1/8 的显存用于 KV Cache），需要通过 PagedAttention 等技术优化 KV Cache 管理。
