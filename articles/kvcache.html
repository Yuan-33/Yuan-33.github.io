KV Cache

在自回归生成中缓存历史 Key-Value 张量，避免重复计算，是提升推理吞吐的核心技术。KV Cache 的管理和优化直接决定了大模型推理系统的性能上限，涉及 PagedAttention、Prefix Caching、KV Cache 压缩等关键优化方向。

基本原理

在 Transformer 自回归生成中，每生成一个新 token 都需要对所有历史 token 计算 Attention。如果不缓存，第 n 步需要重算前 n-1 步的 Key 和 Value，时间复杂度为 O(n²)。KV Cache 将每一步计算出的 Key 和 Value 张量缓存下来，下一步只需计算新 token 的 Q/K/V 并与缓存拼接，将每步复杂度降为 O(n)。

关键技术

  • 基础 KV Cache：缓存每层 Attention 的 Key/Value 张量，避免重复计算历史 token，将推理复杂度从 O(n²) 降至 O(n)
  • PagedAttention（vLLM）：借鉴虚拟内存分页思想，将 KV Cache 划分为固定大小的物理块（Block），按需分配显存，消除预分配导致的内存碎片，显存利用率提升至接近 100%
  • Prefix Caching：对共享相同前缀（如系统提示词）的多个请求，复用同一份 KV Cache，避免重复计算。在多轮对话、Agent 调用等场景下效果显著
  • KV Cache 压缩：GQA（Grouped-Query Attention）和 MQA（Multi-Query Attention）通过共享 Attention Head 减少 KV Cache 大小；H₂O、StreamingLLM 等方法通过注意力分数筛选重要 token 进行保留
  • 显存管理：KV Cache 是大模型推理的主要显存瓶颈。7B 模型处理 4K 上下文约需 1GB KV Cache，70B 模型则需 10GB+。高效的显存调度策略是高并发推理的关键
  • Swap 与 Offloading：当 GPU 显存不足时，可将部分 KV Cache 交换到 CPU 内存或 NVMe SSD，通过异步预取减少延迟影响
